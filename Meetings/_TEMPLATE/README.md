# 議事録作成の準備手順

`_TEMPLATE` は雛形。**コピーして使う。編集しない。**

```text
1. このフォルダをコピーして「YYYYMMDD-会議名」にリネームする
   例：Meetings/20260707-製品会議/

2. MeetingInfo.txt を埋める（会議名・参加メンバー・専用単語）
   ・参加メンバー → 発言者の実名特定に使われる
   ・専用単語（製品型式・企業名など）→ 文字起こしの認識精度が上がる
   ・全会議共通の単語は Meetings/Dictionary.txt に登録すれば毎回書かなくてよい

3. 音声ファイルを入れる（.m4a / .mp3 / .wav そのままでよい）
   ・複数ファイルOK：前半・後半など、フォルダ内の音声はすべて同じ会議として連結処理される
   ・Teams / Zoom の文字起こしがあれば transcript.txt として入れる（音声より優先される）

4. （任意）Memo.txt に覚えている決定事項を走り書きする

5. Claude Code で実行：
   /make-memo YYYYMMDD-会議名

   ※トークン節約：claude --model haiku で起動するか、実行前に /model で Haiku を選ぶとよい
   （文字起こし自体はローカルWhisperなのでトークン消費ゼロ）
```

## 生成される議事録（{フォルダ名}議事録.md）の構成

- **要約版**（前半）：サマリー・決定事項・アクションアイテム・論点別の要旨
- **忠実版**（後半）：時系列の発言記録。発言内容は要約せず忠実に残す

## その他

- 文字起こしはローカルPC内で行われる（音声が外部に送信されることはない）
- 音声ファイルは git にコミットされない（.gitignore で除外済み）
- 決定事項とアクションアイテムは `Meetings/ActionItems.md` に自動集約される
- レコーダーは話者の中央に置くと精度が大きく上がる

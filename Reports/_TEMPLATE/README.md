# レポート作成の準備手順（このフォルダの使い方）

`_TEMPLATE` フォルダは新しいレポートを作るときの雛形。**このフォルダ自体は編集せず、コピーして使う。**

## 展示会・視察レポートの場合

```text
1. このフォルダをコピーして「YYYYMM-イベント名」にリネームする
   例：Reports/202607-TechExpo/

2. Nippou.txt に日報メモを書く（走り書きでよい。これが情報ソースの最優先）

3. Images/ フォルダを作り、写真を全部入れる
   ・リサイズ不要（スキルが横幅800pxに自動変換する）
   ・動画（.MOV/.MP4）もそのまま入れてよい（サムネイル抽出後に自動削除される）

4. Claude Code でリポジトリを開き、実行：
   /build-report YYYYMM-イベント名
```

Report.md の作成・写真の配置・README 目次の更新・KnowledgeBase への知識抽出まで自動で行われる。

## 講演聴講レポートの場合

```text
1. Reports/YYYYMM-イベント名/ フォルダを作る
2. その中に Knowledge/ サブフォルダを作り、以下を入れる：
   ・講演者名-voice.txt （文字起こしテキスト）
   ・Nippou.txt         （日報メモ）
   ・写真（登壇者・会場・配布資料等）
3. Claude Code で「make-lecture のスキルで Lecture.md を作成してください」と依頼する
```

## コツ

- Nippou.txt が充実しているほどレポートの質が上がる。「見た事実」と「感じたこと」を分けて書くとよい
- 写真は撮影日時（タイムスタンプ）を頼りに時系列配置されるので、撮った端末の時刻が正しいことを確認しておく
- 情報が足りない箇所は勝手に創作されず「［要確認：〇〇が不明］」と明記される。あとで埋めて再実行すれば追記される

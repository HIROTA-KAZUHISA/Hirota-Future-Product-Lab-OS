# OUTPUT_STYLE.md — 成果物Markdownの共通整形ルール

すべてのスキル（議事録・販売分析・レポート等）が生成する .md は、このルールに従って **GitHub上で美しく表示される状態** に仕上げる。

## 基本原則

1. **見出しを追うだけで全体がわかる**こと（見出しは内容の要約になっている）
2. **結論・数字が先**、詳細は後
3. GitHub で崩れない記法だけを使う（プレビューで崩れる書き方をしない）

## ドキュメント構造

```markdown
# 絵文字 タイトル（H1は1つだけ）

> **サマリー**：3行以内の要点。読者がここだけ読めば概要がわかる。

| 項目 | 内容 |         ← メタ情報テーブル（日時・場所・出席者・対象期間など）
|---|---|

---                       ← メタ情報と本文の間に区切り線

## 1. セクション（番号＋内容がわかる見出し）
## 2. ...

---                       ← 本文と付録（参考情報・検証メモ等）の間にも区切り線
```

## 見出しの絵文字（統一ルール）

| 用途 | 絵文字 |
|---|:---:|
| 議事録タイトル | 📝 |
| 決定事項 | ✅ |
| アクションアイテム | 📌 |
| 議論・要旨 | 💬 |
| 保留・継続 | ⏸️ |
| 販売分析タイトル | 📊 |
| ハイライト・気づき | 💡 |
| 注意・要確認 | ⚠️ |
| 次回・今後 | ▶️ |

絵文字は **見出し（##）の先頭のみ**。本文中に散らさない。

## テーブル

- 数値・日付・状態の列は `:---:`（中央）または `---:`（右寄せ）で揃える
- 空のテーブルを残さない。項目が無い場合はテーブルではなく「なし」と書く
- 状態表示は絵文字で統一：`🔴 未着手` / `🟡 進行中` / `🟢 完了` / `⚫ 中止`

## アクションアイテム

チェックボックス付きテーブルではなくタスクリスト＋表の併用はしない。表に統一する：

```markdown
| # | アクション | 担当 | 期限 | 状態 |
|:---:|---|---|:---:|:---:|
| 1 | ○○の見積を取る | 廣田 | 7/14 | 🔴 未着手 |
```

## 引用・強調

- サマリー・重要な注記は `> ` blockquote を使う
- 発言の引用は `> 「発言内容」— 発言者` の形式
- 太字は**キーワードのみ**。文全体を太字にしない
- ［要確認：〇〇］［聴取不能］は本文中にそのまま残し、文末の「⚠️ 要確認」セクションにも集約する

## 図・グラフ・写真

```markdown
<br>
<img src="charts/ファイル名.png" width="800">
<p style="color:#888888; font-size:1.05em;">図の読み取り（1〜2行）</p>
```

- 図の前には `<br>` を1行入れる
- 図には必ずグレーのキャプションを付け、「この図から何が言えるか」を書く

## 禁止事項

- H1（#）の複数使用
- 見出しレベルの飛ばし（## の次に ####）
- 空行なしで見出し・表・リストを詰める
- HTMLタグの多用（`<img>` `<br>` `<p>`（キャプション用途）以外は使わない）
- 意味のない箇条書きのネスト（2段まで）
